01

The Baseline Offset Error

Points of Interest (POI) data obtained from open-source maps like Overture frequently drop coordinate pins in the middle of empty parking lots or near the center of massive shopping malls, rather than the true street-facing entrance of the building.

39m Mean Error
124m RMSE
02

Uber H3 Spatial Discretization

To systematically fix this, we discretize the continuous map around each erroneous pin into a local neighborhood of 19 hexagons (K-Ring 2) using Uber's H3 library at Resolution 11 (~20m width).

We convert the repositioning task into a machine learning classification problem: "Which of these 19 surrounding hexagons contains the actual true entrance?"

03

Adding Structural Context

Before classification, the model needs context. We query the Overture Maps API to extract live Building Footprints and Road Segments within the grid, engineering features like the physical area of buildings inside each hexagon or its distance to the nearest roadway.

04

The XGBoost Solution

An XGBoost Machine Learning model ingests this structural context and predicts the most likely hexagon.

By factoring in physical building footprints via this discretized spatial model, TruePin mathematically snaps the inaccurate Overture POIs onto actual building structures, drastically boosting global pinpoint accuracy for delivery optimization.

05

Success Story: Snapping to Structure

Let's look at a concrete example. For this Burlington Outlet Village location, Overture dropped the original pin almost 344 meters away from the true entrance in the middle of a massive parking lot.

By analyzing the structural context within the localized H3 grid, the model successfully identified the correct hexagon containing the building footprint, snapping the pin back to reality.
